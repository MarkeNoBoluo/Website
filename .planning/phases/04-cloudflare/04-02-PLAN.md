<action>
创建首次部署设置脚本，引导用户完成 Cloudflare Tunnel 初始设置。

脚本功能：
1. 检查 cloudflared 是否已安装，如未安装则提供安装指导
2. 引导用户运行 `cloudflared tunnel login`（打开浏览器认证）
3. 引导用户运行 `cloudflared tunnel create blog-tunnel` 生成隧道 ID
4. 指导用户设置环境变量到 .env 文件：
   - `CLOUDFLARE_TUNNEL_ID`（从隧道创建输出中提取）
   - `DOMAIN`（用户输入的域名）
   - `BLOG_SUBDOMAIN`（默认：blog）
   - `STATUS_SUBDOMAIN`（默认：status）
5. 生成初始配置文件：将 config.yml 模板复制到 ~/.cloudflared/ 并替换环境变量
6. 验证配置：运行 `cloudflared tunnel ingress validate`

脚本要求：
- 清晰的用户指导，分步说明
- 错误检查和友好提示
- 不自动执行敏感操作（如隧道创建），仅提供指导
- 生成 .env 文件更新建议

- 首次部署脚本包含生成步骤
- 凭证文件不自动提交到 git
- 环境变量注入配置
