Create `deploy.sh` deployment script with the following functionality:

1. **Script header**:
   - `#!/bin/bash`

   - Add logging function: `log() { echo "$(date): $*"; }`

2. **Configuration variables**:
   - `APP_DIR="/home/pi/blog"`

   - `LOG_DIR="$APP_DIR/logs"`
   - `SERVICE_NAME="blog.service"`
   - `BACKUP_DIR="$APP_DIR/backups"`

3. **Main deployment function** `deploy_main()`:
   - Accept parameters: `$1` = path to checked-out code
   - Create backup of current deployment
   - Copy new code to app directory
   - Run conditional operations based on changed files

4. **Change detection logic**:
   - Function `check_changed_files()`: uses `git diff --name-only OLD_REV NEW_REV`
   - Returns lists: changed_py_files, changed_requirements
   - Also check for other change types: config files, static files, etc.

5. **Conditional operations** (per user decisions):
   - **If requirements.txt changed**: run `pip install -r requirements.txt`
   - **If Python files changed**: restart Flask service (`systemctl restart blog.service`)
   - **If Nginx config changed**: reload Nginx (`systemctl reload nginx`)
   - **If systemd service file changed**: reload systemd and restart service
   - **If only static files changed**: no restart needed

6. **Error handling and logging**:
   - Continue on non-critical errors (log warning)
   - Fail on critical errors (exit with error code)
   - Log all operations to deployment log file
   - Send summary email (optional) or just log

7. **Rollback capability**:
   - Backup current deployment before updating
   - If deployment fails, restore from backup
   - Keep last 3 backups, rotate older ones

8. **Cleanup and notifications**:
   - Remove temporary files
   - Log deployment completion
   - Output deployment summary

Make script executable: `chmod +x deploy.sh`
