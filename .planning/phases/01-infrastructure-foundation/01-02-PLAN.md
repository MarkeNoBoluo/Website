<task type="auto">
  <name>Task 1: Create Gunicorn configuration file</name>
  <files>gunicorn.conf.py</files>
  <action>
Create `gunicorn.conf.py` with the following configuration:

1. **Bind to Unix socket** (not TCP port):
   - `bind = 'unix:/tmp/blog.sock'`
   - Socket will be created at `/tmp/blog.sock`
   - Note: Nginx will connect to this socket

2. **Worker configuration**:
   - `workers = 2` (as specified in requirements)
   - `worker_class = 'sync'` (default, suitable for Flask)
   - `worker_connections = 1000`
   - `timeout = 30` (seconds)
   - `keepalive = 2`

3. **Process naming and user**:
   - `proc_name = 'blog'`
   - `user = 'pi'` (run as pi user per user decision)
   - `group = 'pi'`

4. **Logging configuration**:
   - `accesslog = 'logs/gunicorn_access.log'`
   - `errorlog = 'logs/gunicorn_error.log'`
   - `loglevel = 'info'`
   - `access_log_format = '%(h)s %(l)s %(u)s %(t)s "%(r)s" %(s)s %(b)s "%(f)s" "%(a)s"'`

5. **Process management**:
   - `max_requests = 1000` (restart workers after 1000 requests to prevent memory leaks)
   - `max_requests_jitter = 50` (add randomness to restart timing)
   - `preload_app = True` (load application before forking workers)

6. **Socket permissions**:
   - `umask = 0o007` (socket readable/writable by owner and group)
   - Note: Nginx (www-data user) needs access - will be handled by adding www-data to pi group

7. **Application specification**:
   - `wsgi_app = 'wsgi:app'` (import path for Gunicorn)
   - `chdir = '/home/pi/blog'` (working directory - adjust as needed)

Add comments explaining each setting, especially the Unix socket path and worker count rationale.
  </action>

  </verify>
  <done>Gunicorn configuration file exists with Unix socket binding and 2 workers</done>
