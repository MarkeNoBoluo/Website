Create `nginx/blog.conf` Nginx site configuration:

1. **Basic server setup**:
   - Listen on port 80 (HTTP)
   - Server name: localhost
   - Root directory: `/home/pi/blog/app/static` (for static files)
   - Access and error logs in project logs directory: `~/blog/logs/nginx_*.log`

2. **Gzip compression** (per user decision):
   - Enable gzip: `gzip on`
   - Compress types: text/html, text/css, text/javascript, application/json
   - Compression level: `gzip_comp_level 6`
   - Minimum length: `gzip_min_length 256`

3. **Client limits** (per user decision):
   - `client_max_body_size 10M;` (limit request body size)
   - `client_body_timeout 12s;`
   - `client_header_timeout 12s;`

4. **Static file serving**:
   - Location `/static/` served directly from `/home/pi/blog/app/static`
   - **NO caching headers for static files** (per user decision for simpler debugging)
   - Disable logging for static files to reduce log noise

5. **Reverse proxy to Gunicorn**:
   - Main location `/` proxies to Unix socket
   - `proxy_pass http://unix:/tmp/blog.sock:`
   - Set proper headers:
     - `proxy_set_header Host $host;`
     - `proxy_set_header X-Real-IP $remote_addr;`
     - `proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;`
     - `proxy_set_header X-Forwarded-Proto $scheme;`
   - Timeout settings: `proxy_read_timeout 60s;`

6. **Error pages** (per user decision - use default Nginx error pages):
   - No custom error page configuration
   - Let Nginx use its default error pages

7. **Socket permissions note**:
   - Add comment about socket permissions: Nginx user (www-data) needs access to `/tmp/blog.sock`
   - Solution: `sudo usermod -a -G pi www-data` then restart Nginx

Create directory structure: `mkdir -p nginx/`
Add comments explaining each section, especially the Unix socket proxy and static file serving.
