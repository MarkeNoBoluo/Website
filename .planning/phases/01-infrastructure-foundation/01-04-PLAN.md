<task type="auto">
  <name>Task 1: Create systemd service unit file</name>
  <files>systemd/blog.service</files>
  <action>
Create `systemd/blog.service` systemd unit file:

1. **Unit section**:
   - `Description=Blog Flask Application`
   - `After=network.target` (wait for network per user decision)
   - `Requires=network.target`

2. **Service section**:
   - `Type=simple`
   - `User=pi` (run as pi user per user decision)
   - `Group=pi`
   - `WorkingDirectory=/home/pi/blog`
   - `EnvironmentFile=/home/pi/blog/.env` (load environment variables from file)
   - `ExecStart=/home/pi/blog/.venv/bin/gunicorn -c gunicorn.conf.py`
   - `ExecReload=/bin/kill -s HUP $MAINPID`
   - `ExecStop=/bin/kill -s TERM $MAINPID`
   - `Restart=on-failure` (restart only on failure per user decision)
   - `RestartSec=5s` (wait 5 seconds before restarting)
   - `TimeoutStopSec=30` (30 seconds timeout for graceful shutdown)

3. **Install section**:
   - `WantedBy=multi-user.target` (start on boot)

4. **Logging and permissions**:
   - `StandardOutput=journal`
   - `StandardError=journal`
   - `SyslogIdentifier=blog`
   - `ProtectSystem=strict` (protect system directories)
   - `ReadWritePaths=/home/pi/blog/logs /tmp` (allow writing to logs and temp)

5. **Security hardening** (optional but recommended):
   - `NoNewPrivileges=true`
   - `PrivateTmp=true`
   - `ProtectHome=read-only`
   - `ProtectSystem=strict`

Create directory structure: `mkdir -p systemd/`
Add comments explaining each directive, especially the EnvironmentFile usage and restart policy.
Include instructions for deploying to system:
- Copy to `/etc/systemd/system/blog.service`
- Run `sudo systemctl daemon-reload`
- Run `sudo systemctl enable blog.service`
- Run `sudo systemctl start blog.service`
