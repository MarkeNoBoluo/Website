  </behavior>
  <action>
    1. Add security configuration to app/config.py:
       - Add security-related configuration options with sensible defaults
       - Example:
         ```python
         # Security headers configuration
         SECURITY_HEADERS_ENABLED = True
         SECURITY_CSP_ENABLED = True
         SECURITY_HSTS_ENABLED = False  # Only enable in production with HTTPS
         SECURITY_HSTS_MAX_AGE = 31536000  # 1 year
         SECURITY_CSP_DIRECTIVES = "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline';"
         ```

    2. Create security headers middleware in app/__init__.py:
       - Add after_request handler in create_app() function
       - Apply security headers based on configuration
       - Headers to include:
         * Content-Security-Policy (CSP)
         * Strict-Transport-Security (HSTS) - conditional on HTTPS
         * X-Frame-Options: DENY
         * X-Content-Type-Options: nosniff
         * X-XSS-Protection: 1; mode=block
         * Referrer-Policy: strict-origin-when-cross-origin

    3. Implementation approach:
       - Create `add_security_headers(response)` function
       - Register with `@app.after_request` decorator
       - Check configuration before adding each header
       - For HSTS: only add if SECURITY_HSTS_ENABLED=True and request.is_secure

    4. Configuration via .env:
       - Add security options to .env.example
       - Document each security setting
       - Provide safe defaults for development

    5. Testing considerations:
       - Test that headers are present in responses
       - Test that disabling via config works
       - Ensure headers don't break static file serving (Nginx handles static files)

    Note: Nginx may also add security headers. These Flask headers provide defense-in-depth.
  </action>
